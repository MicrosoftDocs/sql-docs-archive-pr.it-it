## <a name="security-considerations"></a><span data-ttu-id="a038c-129">Considerazioni relative alla sicurezza</span><span class="sxs-lookup"><span data-stu-id="a038c-129">Security Considerations</span></span>

### <a name="protection-of-source-oracle-change-data"></a><span data-ttu-id="a038c-133">Protezione dei dati delle modifiche Oracle di origine</span><span class="sxs-lookup"><span data-stu-id="a038c-133">Protection of Source Oracle Change Data</span></span>

data-ttu-id="a038c-137">I dati acquisiti vengono archiviati nelle istanze di acquisizione di [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] e offrono la stessa protezione delle modifiche [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] database.</span></span> <span data-ttu-id="a038c-138">Per accedere ai dati delle modifiche associati a un'istanza di acquisizione, l'utente  <span data-ttu-id="a038c-141">Questo significa che gli utenti con ruolo predefinito del server **sysadmin** o ruolo predefinito del database **db_owner** dispongono per impostazione predefinita dell'accesso completo ai dati acquisiti ed è possibile concedere ulteriore accesso tramite ruoli di controllo o concedendo l'accesso SELECT alle colonne acquisite.</span><span class="sxs-lookup"><span data-stu-id="a038c-141">This means that users with the **sysadmin** fixed server role or the **db_owner** fixed database role have (by default) full access to the captured data, and further access can be granted either through gating roles or by granting select access to the captured columns.</span></span>

 <span data-ttu-id="a038c-144">La password di log mining viene archiviata crittografata per mezzo di una chiave asimmetrica con il nome `xdbcdc_asym_key` fisso creato automaticamente con il comando seguente:</span><span class="sxs-lookup"><span data-stu-id="a038c-144">The log mining password is stored encrypted by means of an asymmetric key with the fixed name `xdbcdc_asym_key` that is automatically created with the following command:</span></span>
